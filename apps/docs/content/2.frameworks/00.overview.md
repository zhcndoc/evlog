evlog provides native integrations for every major TypeScript framework. The same core API — `log.set()`, `createError()`, `parseError()` — works identically everywhere. Only the setup differs.

## Overview

| Framework | Import | Type | Logger Access | Status |
|-----------|--------|------|---------------|--------|
| [Nuxt](/frameworks/nuxt) | `evlog/nuxt` | Module | `useLogger(event)` | Stable |
| [Next.js](/frameworks/nextjs) | `evlog/next` | Factory | `useLogger()` | Stable |
| [SvelteKit](/frameworks/sveltekit) | `evlog/sveltekit` | Hooks | `event.locals.log` / `useLogger()` | Stable |
| [Nitro](/frameworks/nitro) | `evlog/nitro` | Module | `useLogger(event)` | Stable |
| [TanStack Start](/frameworks/tanstack-start) | `evlog/nitro/v3` | Module | `useRequest().context.log` | Stable |
| [NestJS](/frameworks/nestjs) | `evlog/nestjs` | Module | `useLogger()` | Stable |
| [Express](/frameworks/express) | `evlog/express` | Middleware | `req.log` / `useLogger()` | Stable |
| [Hono](/frameworks/hono) | `evlog/hono` | Middleware | `c.get('log')` | Stable |
| [Fastify](/frameworks/fastify) | `evlog/fastify` | Plugin | `request.log` / `useLogger()` | Stable |
| [Elysia](/frameworks/elysia) | `evlog/elysia` | Plugin | `log` (context) / `useLogger()` | Stable |
| [Cloudflare Workers](/frameworks/cloudflare-workers) | `evlog/workers` | Factory | `createWorkersLogger()` | Stable |
| [Standalone](/frameworks/standalone) | `evlog` | Manual | `createLogger()` / `createRequestLogger()` | Stable |
| [Astro](/frameworks/astro) | `evlog` | Manual | `createRequestLogger()` | Guide |
| [Custom](/frameworks/custom-integration) | `evlog/toolkit` | Build your own | `createMiddlewareLogger()` | Beta |
