`evlog/better-auth` 将匿名宽事件转化为已识别事件。每个请求自动包含发起者——无需手动 `log.set({ user })`。

::code-collapse

```txt [提示]
在我的应用中添加带有 evlog 的 Better Auth 用户识别。

- 从 'evlog/better-auth' 导入 createAuthMiddleware
- 调用 createAuthMiddleware(auth) 获取 identify 函数
- 在中间件/钩子中调用 identify(log, headers, path) 以自动识别每个请求的用户
- 默认安全——仅提取白名单字段，从不记录密码或令牌
- 支持包含/排除路由模式、生命周期钩子和 Better Auth 插件字段
- 适用于所有框架：Nuxt、Next.js、Express、Hono、Fastify、NestJS、Elysia、独立模式

文档：https://www.evlog.dev/logging/better-auth
适配器：https://www.evlog.dev/adapters
```

::

## 快速开始

一个中间件，所有请求均被识别：

::code-group
```typescript [server/middleware/auth-identify.ts (Nuxt)]
import { createAuthMiddleware } from 'evlog/better-auth'

const identify = createAuthMiddleware(auth, {
  exclude: ['/api/auth/**'],
})

export default defineEventHandler(async (event) => {
  if (!event.context.log) return
  await identify(event.context.log, event.headers, event.path)
})
```

```typescript [app/api/checkout/route.ts (Next.js)]
import { withEvlog, useLogger } from '@/lib/evlog'
import { createAuthMiddleware } from 'evlog/better-auth'
import { auth } from '@/lib/auth'

const identify = createAuthMiddleware(auth)

export const POST = withEvlog(async (request: Request) => {
  const log = useLogger()
  await identify(log, request.headers)
  log.set({ action: 'checkout' })
  return Response.json({ success: true })
})
```

```typescript [src/index.ts (Express)]
import { createAuthMiddleware } from 'evlog/better-auth'

const identify = createAuthMiddleware(auth, {
  exclude: ['/api/auth/**'],
})

app.use(async (req, res, next) => {
  await identify(req.log, req.headers, req.path)
  next()
})
```

```typescript [src/index.ts (Hono)]
import { createAuthMiddleware } from 'evlog/better-auth'

const identify = createAuthMiddleware(auth, {
  exclude: ['/api/auth/**'],
})

app.use(async (c, next) => {
  await identify(c.get('log'), c.req.raw.headers, c.req.path)
  await next()
})
```

```typescript [src/index.ts (Fastify)]

  exclude: ['/api/auth/**'],
})

```

```typescript [src/index.ts (Elysia)]
import { createAuthMiddleware } from 'evlog/better-auth'

const identify = createAuthMiddleware(auth, {
  exclude: ['/api/auth/**'],
})

app.derive(async ({ log, request }) => {
  await identify(log, request.headers, new URL(request.url).pathname)
  return {}
})
```

```typescript [src/auth-identify.middleware.ts (NestJS)]
import { createAuthMiddleware } from 'evlog/better-auth'
import { useLogger } from 'evlog/nestjs'

const identify = createAuthMiddleware(auth, {
  exclude: ['/api/auth/**'],
})

@Injectable()
export class AuthIdentifyMiddleware implements NestMiddleware {
  async use(req: Request, res: Response, next: NextFunction) {
    await identify(useLogger(), req.headers, req.path)
    next()
  }
}
```

```typescript [scripts/sync-job.ts (独立模式)]
import { identifyUser } from 'evlog/better-auth'
import { createLogger } from 'evlog'

const log = createLogger()
const session = await auth.api.getSession({ headers })
if (session) identifyUser(log, session)
log.emit()
```

## 工作原理

集成从请求 cookie 中解析 Better Auth 会话，提取安全的用户和会话字段，并将其设置到记录器上。默认情况下跳过认证路由。

`请求` → `中间件` → `Better Auth` → `记录器` → `排出`

1. 传入请求命中你的中间件
2. 中间件检查 `include`/`exclude` 模式——如果路由不匹配则跳过
3. `getSession(headers)` 通过 Better Auth 解析会话（捕获耗时）
4. `identifyUser(log, session)` 在记录器上设置 `userId`、`user` 和 `session`
5. 根据会话结果触发 `onIdentify` 或 `onAnonymous` 钩子
6. 请求结束时，宽事件连同完整的用户上下文一起发射到你的 drain

| 导出 | 描述 |
|--------|-------------|
| `identifyUser(log, session)` | 核心辅助函数——从会话中提取安全字段并将其设置到记录器上。如果已识别则返回 `true` |
| `createAuthMiddleware(auth)` | 返回一个异步 `(log, headers, path?) => Promise<boolean>` 函数，包含路由过滤、计时和钩子 |
| `createAuthIdentifier(auth)` | 用于独立 Nitro 应用的 Nitro `request` 钩子工厂 |
| `maskEmail(email)` | 掩码邮箱：`hugo@example.com` → `h***@example.com` |

## identifyUser

核心构建块。接受一个 `RequestLogger` 和一个 Better Auth 会话，提取安全字段，并调用 `log.set()`。如果用户被识别则返回 `true`，否则返回 `false`：

```typescript [server/api/checkout.post.ts]
import { identifyUser } from 'evlog/better-auth'

const session = await auth.api.getSession({ headers: event.headers })
if (session) {
  const identified = identifyUser(log, session)
  if (identified) {
    log.set({ subscription: 'premium' })
  }
}
```

默认安全——仅提取白名单字段。密码、令牌和密钥永远不会被包含。

### 选项

| 选项 | 类型 | 默认值 | 描述 |
|--------|------|---------|-------------|
| `maskEmail` | `boolean` | `false` | 将邮箱掩码为 `h***@example.com` |
| `session` | `boolean` | `true` | 包含会话元数据（`session.id`、`session.expiresAt`、`session.ipAddress`、`session.userAgent`） |
| `fields` | `string[]` | `['id', 'name', 'email', 'image', 'emailVerified', 'createdAt']` | 要提取的用户字段 |
| `extend` | `(session) => Record<string, unknown>` | `undefined` | 从 Better Auth 插件添加自定义字段（组织、角色等） |

```typescript [server/api/checkout.post.ts]
identifyUser(log, session, {
  maskEmail: true,
  fields: ['id', 'name'],
  session: false,
})
```

### extend

使用 `extend` 捕获由 Better Auth 插件添加的字段（组织、2FA、角色等）：

```typescript [server/middleware/auth-identify.ts]
import { createAuthMiddleware } from 'evlog/better-auth'

const identify = createAuthMiddleware(auth, {
  extend: (session) => ({
    organization: session.user.activeOrganization,
    role: session.user.role,
  }),
})
```

带有插件字段的宽事件：

```json [宽事件]
{
  "userId": "QBX9tPjJQExWawAbNll75",
  "user": { "id": "QBX9tPjJQExWawAbNll75", "name": "Hugo Richard" },
  "organization": { "id": "org_42", "name": "Acme" },
  "role": "admin"
}
```

## createAuthMiddleware

框架无关的工厂。在启动时调用一次，然后在中间件中使用返回的函数。第三个参数 `path` 启用内置路由过滤：

```typescript [server/middleware/auth-identify.ts]
import { createAuthMiddleware } from 'evlog/better-auth'

const identify = createAuthMiddleware(auth, {
  exclude: ['/api/auth/**', '/api/public/**'],
  include: ['/api/**'],
  maskEmail: true,
})
```

函数签名为 `(log, headers, path?) => Promise<boolean>`。它解析会话，调用 `identifyUser`，捕获计时，触发生命周期钩子，并静默捕获错误，因此会话解析永远不会破坏请求。

### 选项

继承所有 `identifyUser` 选项，外加：

| 选项 | 类型 | 默认值 | 描述 |
|--------|------|---------|-------------|
| `exclude` | `string[]` | `['/api/auth/**']` | 要跳过的路由模式（glob） |
| `include` | `string[]` | `undefined` | 如果设置，则仅解析匹配的路由 |
| `onIdentify` | `(log, session) => void` | `undefined` | 成功识别后调用 |
| `onAnonymous` | `(log) => void` | `undefined` | 未找到会话时调用 |

### 生命周期钩子

使用 `onIdentify` 响应用户识别——例如，通过尾部采样强制保留高级用户的日志：

```typescript [server/middleware/auth-identify.ts]
const identify = createAuthMiddleware(auth, {
  onIdentify: (log, session) => {
    if (session.user.plan === 'enterprise') {
      log.set({ _forceKeep: true })
    }
  },
  onAnonymous: (log) => {
    log.set({ anonymous: true })
  },
})
```

## createAuthIdentifier (独立 Nitro)

创建一个 Nitro `request` 钩子的工厂。专为独立 Nitro 应用设计，其中 evlog Nitro 模块处理钩子排序。

::note
对于 **Nuxt**，请在服务器中间件中使用 `createAuthMiddleware` —— Nitro 插件钩子排序可能导致 `request` 钩子中记录器尚不可用。
::

```typescript [server/plugins/evlog-auth.ts]
import { createAuthIdentifier } from 'evlog/better-auth'
import { auth } from './lib/auth'

export default defineNitroPlugin((nitroApp) => {
  nitroApp.hooks.hook('request', createAuthIdentifier(auth, {
    exclude: ['/api/auth/**', '/api/public/**'],
  }))
})
```

## 性能

`getSession()` 在每个请求上消耗一次数据库查询。宽事件中的 `auth.resolvedIn` 字段告诉你每个解析确切耗时多久。对于高流量应用：

1. **在 Better Auth 中启用 [会话缓存](https://www.better-auth.com/docs/concepts/session-caching)** 以避免在每个请求上命中数据库
2. **使用 `exclude`** 跳过不需要用户上下文的公共路由
3. **使用 `include`** 将解析限制为特定路由模式

```json [宽事件 — 慢会话解析]
{
  "auth": { "resolvedIn": 245, "identified": true },
  "duration": "312ms"
}
```

当 `auth.resolvedIn` 相对于 `duration` 较高时，在 Better Auth 中启用会话缓存。

## 客户端身份同步

在客户端，监听 Better Auth 会话并调用 `setIdentity()` 以在客户端日志中包含用户上下文：

```typescript [composables/useAuthIdentity.ts]
import { authClient } from '~/lib/auth-client'

export function useAuthIdentity() {
  const session = authClient.useSession()

  watch(() => session.value?.data?.user, (user) => {
    if (user) {
      setIdentity({ userId: user.id, userName: user.name })
    } else {
      clearIdentity()
    }
  }, { immediate: true })
}
```

在你的根布局中调用一次：

```vue [app.vue]
<script setup>
useAuthIdentity()
</script>
```

客户端日志现在包含用户身份：

```json [客户端日志]
{
  "level": "info",
  "tag": "checkout",
  "message": "用户点击了结账",
  "userId": "QBX9tPjJQExWawAbNll75",
  "userName": "Hugo Richard"
}
```

## 宽事件字段

| 字段 | 来源 | 描述 |
|-------|--------|-------------|
| `userId` | `session.user.id` | 顶层用户 ID（由 PostHog 适配器用作 `distinct_id`） |
| `user.id` | `session.user.id` | 用户 ID |
| `user.name` | `session.user.name` | 显示名称 |
| `user.email` | `session.user.email` | 邮箱（可使用 `maskEmail: true` 掩码） |
| `user.image` | `session.user.image` | 头像 URL |
| `user.emailVerified` | `session.user.emailVerified` | 邮箱验证状态 |
| `user.createdAt` | `session.user.createdAt` | 账户创建日期（ISO 字符串） |
| `session.id` | `session.session.id` | 会话 ID |
| `session.expiresAt` | `session.session.expiresAt` | 会话过期时间（ISO 字符串） |
| `session.ipAddress` | `session.session.ipAddress` | 来自会话的客户端 IP |
| `session.userAgent` | `session.session.userAgent` | 来自会话的用户代理字符串 |
| `session.createdAt` | `session.session.createdAt` | 会话创建日期（ISO 字符串） |
| `auth.resolvedIn` | 测量 | 会话解析时间（毫秒） |
| `auth.identified` | 计算 | 请求是否已被识别 |
