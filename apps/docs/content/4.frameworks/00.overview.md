evlog 为每一个主要的 TypeScript 框架提供原生集成。相同的核心 API（`log.set()`、`createError()`、`parseError()`）在所有环境中表现一致。唯一的区别在于设置方式。

## 概述

| 框架 | 导入 | 类型 | 日志访问方式 | 状态 |
|------|------|------|--------------|------|
| [Nuxt](/frameworks/nuxt) | `evlog/nuxt` | 模块 | `useLogger(event)` | 稳定 |
| [Next.js](/frameworks/nextjs) | `evlog/next` | 工厂 | `useLogger()` | 稳定 |
| [SvelteKit](/frameworks/sveltekit) | `evlog/sveltekit` | 钩子 | `event.locals.log` / `useLogger()` | 稳定 |
| [Nitro](/frameworks/nitro) | `evlog/nitro` | 模块 | `useLogger(event)` | 稳定 |
| [TanStack Start](/frameworks/tanstack-start) | `evlog/nitro/v3` | 模块 | `useRequest().context.log` | 稳定 |
| [TanStack Router](/frameworks/tanstack-start) | `evlog/nitro/v3` | 模块 | 通过 TanStack Start（使用 Nitro v3） | 稳定 |
| [React Router](/frameworks/react-router) | `evlog/react-router` | 中间件 | `context.get(loggerContext)` / `useLogger()` | 稳定 |
| [NestJS](/frameworks/nestjs) | `evlog/nestjs` | 模块 | `useLogger()` | 稳定 |
| [Express](/frameworks/express) | `evlog/express` | 中间件 | `req.log` / `useLogger()` | 稳定 |
| [Hono](/frameworks/hono) | `evlog/hono` | 中间件 | `c.get('log')` | 稳定 |
| [Fastify](/frameworks/fastify) | `evlog/fastify` | 插件 | `request.log` / `useLogger()` | 稳定 |
| [Elysia](/frameworks/elysia) | `evlog/elysia` | 插件 | `log`（上下文） / `useLogger()` | 稳定 |
| [Cloudflare Workers](/frameworks/cloudflare-workers) | `evlog/workers` | 工厂 | `createWorkersLogger()` | 稳定 |
| [AWS Lambda](/frameworks/aws-lambda) | `evlog` | 手动 | `createLogger()` / `createRequestLogger()` | 指南 |
| [独立使用](/frameworks/standalone) | `evlog` | 手动 | `createLogger()` / `createRequestLogger()` | 稳定 |
| [Astro](/frameworks/astro) | `evlog` | 手动 | `createRequestLogger()` | 指南 |
| [自定义集成](/frameworks/custom-integration) | `evlog/toolkit` | 自行构建 | `createMiddlewareLogger()` | 测试版 |

## Vite 插件

对于任意基于 Vite 的项目，[`evlog/vite` 插件](/core-concepts/vite-plugin) 提供构建时优化：

- **自动初始化**：无需手动调用 `initLogger()`
- **调试剥离**：生产环境中自动移除 `log.debug()`
- **源码位置**：在日志调用中注入 `__source: 'file:line'`

适用于 SvelteKit、Hono（通过 vite-node）以及任意 Vite 驱动的配置。Nuxt 用户可通过 `evlog/nuxt` 模块选项获得这些功能。
