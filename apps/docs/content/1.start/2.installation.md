在我的 TypeScript 项目中安装 evlog。

- 检测我正在使用的框架（Nuxt、Next.js、SvelteKit、Nitro、NestJS、Express、Hono、Fastify、Elysia、TanStack Start、React Router、Cloudflare Workers，或独立项目）
- 使用我的包管理器安装 evlog：pnpm add evlog（或 npm/yarn/bun）
- 配置特定于框架的集成（模块、插件或中间件）
- 将 evlog.env.service 设置为我的应用名称
- 确认 useLogger、createError 和 parseError 可用

Docs: https://www.evlog.dev/start/installation
Frameworks: https://www.evlog.dev/integrate/frameworks/overview

::

## 安装

::code-group
```bash [pnpm]
pnpm add evlog
```
```bash [bun]
bun add evlog
```
```bash [yarn]
yarn add evlog
```
```bash [npm]
npm install evlog
```
::

### 使用 Agent 技能

如果你使用了 AI 助手（如 Claude Code、Cursor 等），安装 evlog 技能以获得引导式设置和代码审查：

```bash [Terminal]
npx skills add https://www.evlog.dev
```

Your AI assistant can then help you set up evlog, review your logging patterns, and migrate existing code to wide events. See [Agent Skills](/reference/agent-skills) for details.

## TypeScript 配置

evlog 附带完整的 TypeScript 类型定义，无需额外配置。

::callout{icon="i-lucide-check" color="success"}
evlog 需要 TypeScript 5.0 或更高版本以获得最佳的类型推断。
