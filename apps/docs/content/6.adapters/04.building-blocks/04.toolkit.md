`evlog/toolkit` 是每个内置适配器、增强器和框架集成所构建于其上的公共表面。如果你正在基于 evlog 发布一个社区包，这里就是你的入口点。

::callout{icon="i-lucide-flask-conical" color="warning"}
工具包标记为 **beta**。其形状是稳定的（所有内置功能都在生产中使用），并遵循 semver —— 但在社区使用验证该表面之前，仍然可能发生破坏性变更。
::

## 插件契约

`definePlugin` 是规范的扩展契约。Drains 和 enrichers 都是对它的语法糖。

```typescript
import { definePlugin } from 'evlog/toolkit'

const requestMetricsPlugin = definePlugin({
  name: 'request-metrics',

  setup({ env }) {
    statsd.init({ service: env.service })
  },

  enrich({ event }) {
    event.tier = event.duration && event.duration > 1000 ? 'slow' : 'fast'
  },

  drain({ event }) {
    statsd.timing('http.request', event.duration as number, { path: event.path as string })
  },

  onRequestStart({ logger, request }) {
    logger.set({ trace: { startedAt: Date.now() } })
  },

  onRequestFinish({ event, durationMs }) {
    if (event && (event.level === 'error' || durationMs > 5000)) {
      // 告警 / 转发 / 等等
    }
  },

  onClientLog({ event }) {
    // 接入通过 /api/_evlog/ingest 接收的客户端日志
  },

  extendLogger(logger) {
    // 在这里为 RequestLogger 添加自定义的类型化方法
  },
})
```

通过 `defineEvlog({ plugins: [requestMetricsPlugin] })` 一次性注册，或通过 `evlog({ plugins: [requestMetricsPlugin] })` 按中间件范围注册。插件按注册顺序运行；任何钩子中的错误都会被隔离并以 `[evlog/<name>]` 前缀记录。

### 语法糖插件

```typescript
import { drainPlugin, enricherPlugin } from 'evlog/toolkit'

const drainOnly = drainPlugin('axiom', createAxiomDrain())
const enricherOnly = enricherPlugin('user-agent', createUserAgentEnricher())
```

## `defineHttpDrain` —— 适配器配方

```typescript
import {
  defineHttpDrain,
  resolveAdapterConfig,
  type ConfigField,
} from 'evlog/toolkit'

interface AcmeConfig {
  apiKey: string
  endpoint?: string
  timeout?: number
}

const FIELDS: ConfigField<AcmeConfig>[] = [
  { key: 'apiKey', env: ['ACME_API_KEY'] },
  { key: 'endpoint', env: ['ACME_ENDPOINT'] },
  { key: 'timeout' },
]

export function createAcmeDrain(overrides?: Partial<AcmeConfig>) {
  return defineHttpDrain<AcmeConfig>({
    name: 'acme',
    resolve: async () => {
      const cfg = await resolveAdapterConfig<AcmeConfig>('acme', FIELDS, overrides)
      return cfg.apiKey ? cfg as AcmeConfig : null
    },
    encode: (events, cfg) => ({
      url: `${cfg.endpoint ?? 'https://api.acme.com'}/v1/ingest`,
      headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${cfg.apiKey}` },
      body: JSON.stringify(events),
    }),
  })
}
```

`defineHttpDrain` 负责批处理、重试（默认 2 次）、超时（默认 5000ms）和错误隔离。

## `defineEnricher` —— 增强器配方

```typescript
import { defineEnricher, getHeader } from 'evlog/toolkit'

export const tenantEnricher = defineEnricher<{ id: string }>({
  name: 'tenant',
  field: 'tenant',
  compute: ({ headers }) => {
    const id = getHeader(headers, 'x-tenant-id')
    return id ? { id } : undefined
  },
})
```

## `defineEvlog` —— 规范配置

一个可在各处通用的单一配置对象——`initLogger`、框架中间件、Nuxt 模块、Workers。

```typescript
import { defineEvlog, toLoggerConfig, toMiddlewareOptions } from 'evlog/toolkit'

export const evlogConfig = defineEvlog({
  service: 'shop',
  environment: process.env.NODE_ENV,
  drain: createAxiomDrain(),
  enrich: createDefaultEnrichers(),
  plugins: [requestMetricsPlugin],
})

// 独立使用
initLogger(toLoggerConfig(evlogConfig))

// 框架中
app.use(evlog(toMiddlewareOptions(evlogConfig)))
```

## 组合

```typescript
import { composeDrains, composeEnrichers, composeKeep } from 'evlog/toolkit'

const drain = composeDrains([createAxiomDrain(), createSentryDrain()])
const enrich = composeEnrichers([createUserAgentEnricher(), createGeoEnricher()])
const keep = composeKeep([
  ({ duration, shouldKeep }) => duration && duration > 2000 ? true : shouldKeep,
  ({ event }) => event.level === 'error',
])
```

所有组合器都会隔离单个函数中的错误，并使用 `Promise.allSettled` 语义并发运行 drains。

## 帮助函数

| 导出 | 作用 |
|--------|---------|
| `httpPost(opts)` | 每个内置 HTTP 适配器都使用的 POST 帮助函数——处理超时、重试、脱敏错误消息 |
| `resolveAdapterConfig(ns, fields, overrides)` | 标准配置优先级：overrides → `runtimeConfig.evlog.<ns>` → `runtimeConfig.<ns>` → `NUXT_<NS>_*` → `<NS>_*` |
| `getHeader(headers, name)` | 不区分大小写的 HTTP 请求头查找 |
| `normalizeNumber(value)` | 将字符串解析为数字；若不是有限数则返回 `undefined` |
| `extractSafeHeaders(headers)` | 从 Web `Headers` 中过滤敏感请求头 |
| `extractSafeNodeHeaders(headers)` | 从 Node `IncomingHttpHeaders` 中过滤敏感请求头 |
| `mergeEventField(existing, computed, overwrite?)` | 将子对象合并到事件字段中，并遵循 `overwrite` |
| `toTypedAttributeValue(value)` | 将任意值转换为 Axiom / Sentry 使用的类型化属性形状 |
| `toOtlpAttributeValue(value)` | 将任意值转换为 OTLP `AnyValue` 形状（用于 OTLP / HyperDX / PostHog 日志） |
| `OTEL_SEVERITY_NUMBER`, `OTEL_SEVERITY_TEXT` | OTEL 日志严重级别表 |

## 构建社区包

建议的 evlog 社区包结构：

```
my-evlog-pkg/
├─ src/
│  ├─ drain.ts        # 通过 defineHttpDrain 创建 createMyDrain
│  ├─ enricher.ts     # 通过 defineEnricher 创建 createMyEnricher
│  └─ index.ts        # 再导出
├─ test/              # vitest，mock fetch
├─ package.json       # peerDependency: "evlog"
└─ README.md
```

将 `evlog` 添加为 `peerDependency`（而不是 dependency）——你的包不应该在安装时拉入一份 evlog 副本。
